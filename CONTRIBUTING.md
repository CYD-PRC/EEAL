# Contributing to EEAL

## How to contribute

1. Fork this repository
2. Create a branch
3. Commit your changes
4. Push to your fork
5. Open a Pull Request

## What to contribute

- Protocol specification improvements
- Reference implementations in any language
- Bug reports and feature requests
- Documentation and examples
- Industry use cases

## Guidelines

- All protocol changes must include rationale
- All claims must be supported by verifiable evidence (test logs / CI reports / data files)

## Discussions

Open an issue for any questions or suggestions.
